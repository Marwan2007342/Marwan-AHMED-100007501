Ink Marker Manufacturing Production Line

A simulated factory cell for molding marker barrels, filling ink reservoirs, installing tips, assembling caps, and rejecting defective units through automated quality checks.

Prepared By Marwan Ahmed Student ID 100007510
Route Map

One product advances through five sequential stations. A fault stops the cycle and records the defect.

  1. 01

    Plastic Barrel Molding

    Forms the marker shell and checks for flash, warping, and wall-thickness errors.

    3.0s / 12%
  2. 02

    Ink Reservoir Filling

    Meters ink into the reservoir and validates volume, viscosity, and leakage.

    4.0s / 10%
  3. 03

    Marker Tip Installation

    Seats the fiber tip, then checks depth, alignment, and channel flow.

    2.5s / 8%
  4. 04

    Cap Assembly

    Fits the cap and verifies clip shape, snap force, and seal engagement.

    2.0s / 7%
  5. 05

    Quality Control

    Inspects writing flow, cap seal, dimensions, and external finish before release.

    3.5s / 6%
Control Architecture

Separated modules, one operator view.

The Python backend owns the state machine and defect logic. The Pygame HMI renders the line status and operator controls. InfluxDB receives metric snapshots through a background writer, while Grafana reads the marker_production measurement.

Backend ProductionLine, Product, Machine, QualityControl
HMI Start, stop, reset, emergency stop, stage status cards
Telemetry parts_produced, defects_total, current_stage, active_fault
Monitoring Provisioned dashboard and datasource through Docker Compose
